# 엘보우(Elbow) 적용을 통한 라우팅 오차 개선 및 기존 설계 모사 방안 분석

기존 설계 데이터(`TB_ROUTE_SEGMENT_DETAIL`)로부터 배관 경로를 복원하고 자동 라우팅을 진행할 때, 엘보우(Elbow) 구간이 **사선 배관(Diagonal Pipe)**으로 오인되어 발생하는 문제점을 분석하고, 이를 직교 기하 구조의 **꺾임점(Intersection Point, IP)**으로 변환하여 해결하는 개선 방식을 제안합니다.

---

## 1. 기존 방식의 문제점 분석

기존 설계 데이터에서 엘보우는 자재의 입구 포트(FROM_POS)와 출구 포트(TO_POS)가 3D 공간 상의 두 점으로 저장됩니다.

### 1.1 기하학적 왜곡 (사선 배관 오인)
* **현상**: X축 방향으로 진행하던 배관이 Y축 방향으로 꺾일 때, 90도 엘보우 자재의 양 끝점인 `P_in`과 `P_out`은 X와 Y 좌표가 모두 다릅니다. 이 두 점을 단순 직선으로 이으면 3D 공간에서 45도 방향의 **사선 세그먼트**가 생성됩니다.
* **영역별 영향**:
  * **라우팅 알고리즘**: 라우터가 해당 구간을 직교 배관이 아닌 사선(Diagonal) 배관 경로로 인식하여, 불필요한 사선 탐색을 시도하거나 격자 매핑 시 미세한 위치 진동(Jitter)을 유발합니다.
  * **패턴 분석 모델 (`ExtractStubPatterns` 등)**: 배관의 토폴로지 방향열(Direction Sequence)을 추출할 때 `+x -> +y`가 아니라 `+x -> 사선 -> +y`와 같이 왜곡된 방향 패턴으로 분석되어 특징 벡터(Feature Vector)의 품질이 저하됩니다.

```mermaid
graph TD
    subgraph 현재 방식: 사선 연결 (오류 발생)
        A1[직관: X축 방향] -->|P_in| B1(엘보우: X/Y 사선 세그먼트)
        B1 -->|P_out| C1[직관: Y축 방향]
    end
    
    subgraph 개선 방식: 직교화 (IP 복원)
        A2[직관: X축 방향] -->|P_in| IP[가상 교차점 IP]
        IP -->|P_out| C2[직관: Y축 방향]
    end
```

### 1.2 특징점(높이, 꺾임점) 좌표 오차
* **현상**: 실제 배관 설계에서 꺾임점의 기준은 엘보우의 외곽 점이 아닌 두 직선의 가상 교차점(IP)입니다. 
* **오차 원인**: 기존 프로그램에서는 엘보우의 위치를 단순히 두 포트의 중점 `(P_in + P_out) * 0.5`로 잡거나 두 포트 중 하나를 꺾임점으로 취급합니다. 이로 인해 배관 규격(Size) 및 엘보우 종류(Short/Long Radius)에 따라 **수십~수백 mm의 기하학적 오차(높이, 꺾임 위치)**가 발생하며 기존 설계 데이터와 정밀한 정렬이 불가능해집니다.

---

## 2. 개선 방식 제안

기존 설계와의 기하학적 유사성을 극대화하고 오차를 최소화하기 위해 세 가지 단계별 개선 방식을 제시합니다.

```
                  [가상 교차점 IP (Intersection Point)]
                                   + (X2, Y1, Z1)
                                  / \
                                 /   \ 
                                /     \
                               /       \
      ========================+         +========================
         직선 배관 (X축)      P_in       P_out      직선 배관 (Y축)
         <--- Center-to-End (E) --->|<--- Center-to-End (E) --->
```

### 개선안 A: 이전/이후 직선 배관 방향을 이용한 IP 기하 복원 (Reconstruction)
기존 DB의 데이터를 정제(Preprocessing)하여 폴리라인을 복원할 때, 엘보우 세그먼트를 감지하고 직교 교차점을 수학적으로 계산하여 대체하는 방식입니다.

1. **엘보우 감지**: `TB_ROUTE_SEGMENT_DETAIL`을 Order 순으로 스캔하며 `TYPE = 'ELBOW'`인 세그먼트를 식별합니다.
2. **입/출력 벡터 추출**: 엘보우 입구 포트 `P_in`에 연결된 이전 직관의 진행 방향 벡터 $\vec{V}_{in}$과 출구 포트 `P_out`에 연결된 다음 직관의 진행 방향 벡터 $\vec{V}_{out}$을 구합니다.
3. **가상 교차점(IP) 계산**: 
   * 두 직선 $L_1(t) = P_{in} + t \cdot \vec{V}_{in}$ 과 $L_2(s) = P_{out} + s \cdot \vec{V}_{out}$ 이 만나는 교차점 $P_{IP}$를 구합니다.
   * 직교 좌표계(Orthogonal) 환경에서 두 축이 직각으로 교차하는 경우, 교차점 좌표는 각 축의 성분을 결합하여 쉽게 계산할 수 있습니다.
     * *예: X축 진입 후 Y축 진출 시, $P_{IP} = (P_{out}.X, P_{in}.Y, P_{in}.Z)$*
4. **폴리라인 대체**: 복원된 배관 중심선 점열에서 사선 세그먼트 `[P_in, P_out]` 대신 `[P_in, P_IP, P_out]` 또는 단순히 이전 직관을 $P_{IP}$까지 연장한 `[..., P_IP, ...]` 형태로 가공합니다.

---

### 개선안 B: 엘보우 규격(Center-to-End) 데이터 테이블 기반 정밀 역산
엘보우 규격(BIM/피팅 라이브러리 치수) 정보를 활용하여 정확한 꺾임점을 산출하는 방식입니다.

1. **규격 데이터베이스 구축**: 배관 사이즈(Diameter/Size) 및 엘보우 각도(90도, 45도 등)에 따른 **Center-to-End 거리(E)** 정보를 매핑 테이블로 정의합니다.
2. **IP 좌표 역산**:
   * 엘보우의 입구 `P_in`에서 진입 방향으로 $E$만큼 이동하고, 출구 `P_out`에서 진출 반대 방향으로 $E$만큼 이동하여 가상 교차점 IP를 정의합니다.
   * $$P_{IP} = P_{in} + E \cdot \vec{V}_{in} = P_{out} - E \cdot \vec{V}_{out}$$
3. **효과**: 배관 사이즈에 따라 달라지는 엘보우 크기를 정확하게 반영하므로, 높이 및 꺾임점에 대한 수학적 오차가 $0\text{mm}$에 가깝게 수렴합니다.

---

### 개선안 C: 라우팅 코어 직교 탐색 및 포스트 피팅(Post-fitting) 분리
자동 라우팅 프로그램의 탐색 로직 자체를 단순화하고 결합도를 낮추는 아키텍처 개선안입니다.

```mermaid
seqContainer:
    Note over 라우터: 1. 직교 격자 탐색 (A* 등)
    라우터->>결과 경로: 2. 꺾임점 (IP) 좌표 도출
    Note over 피팅 엔진: 3. IP 기준 Center-to-End (E) 거리만큼 이격
    피팅 엔진->>기존 DB 포맷: 4. [PIPE] - [ELBOW (P_in~P_out)] - [PIPE] 분할 저장
```

1. **경로 탐색 단계**: 라우터 엔진은 공간을 직교 격자망으로만 해석하고, 사선 경로를 배제한 채 직교 교차점($P_{IP}$)들을 연결하는 직각 경로를 찾습니다.
2. **자재 분할 후처리 단계**: 탐색된 직교 경로에서 꺾임점($P_{IP}$)을 기준으로 배관 사이즈별 엘보우 규격 $E$만큼 이전 방향과 이후 방향으로 후퇴한 지점 $P_{in}$과 $P_{out}$을 계산합니다.
3. **결과 저장**:
   * $P_{in}$ 이전까지는 `TYPE = 'PIPE'` (직선 배관)
   * $P_{in} \rightarrow P_{out}$ 구간은 `TYPE = 'ELBOW'` (엘보우 자재)
   * $P_{out}$ 이후부터는 다음 `TYPE = 'PIPE'` (직선 배관)로 분할하여 DB에 기록합니다.
